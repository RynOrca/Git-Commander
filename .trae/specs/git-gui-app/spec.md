# Git Commander 图形化Git操作应用规格说明

## 为什么创建这个应用
许多开发者和非技术人员想要使用Git进行版本控制，但被复杂的命令行操作所阻碍。这个应用将通过直观的图形界面，让不懂Git命令的用户也能轻松进行版本控制操作。

## 目标用户
- 不熟悉命令行操作的Git初学者
- 需要简化Git工作流程的团队成员
- 设计师、产品经理等非开发岗位人员

## 应用概述
Git Commander是一款Windows桌面应用，提供可视化的Git操作界面，让用户通过点击按钮即可完成常用的Git操作，无需记忆复杂的命令。

## 核心功能

### 1. 仓库管理
- **打开本地仓库**：选择本地已有的Git仓库文件夹
- **创建新仓库**：在指定目录初始化新的Git仓库
- **最近打开记录**：显示最近打开的仓库列表

### 2. 文件状态可视化
- **未跟踪文件（Untracked）**：显示新增但未被Git管理的文件
- **已修改文件（Modified）**：显示已修改但未暂存的文件
- **已暂存文件（Staged）**：显示已暂存准备提交的文件
- **文件差异查看**：点击文件查看具体修改内容

### 3. 提交（Commit）操作
- **暂存文件**：勾选要提交的文件，一键暂存
- **填写提交信息**：输入提交说明（标题+详细描述）
- **执行提交**：点击按钮完成提交

### 4. 远程仓库操作
- **克隆仓库**：通过URL克隆远程仓库到本地
- **推送（Push）**：将本地提交推送到远程仓库
- **拉取（Pull）**：从远程仓库拉取最新代码
- **获取（Fetch）**：获取远程更新但不合并

### 5. 分支管理
- **查看分支列表**：显示所有本地和远程分支
- **创建分支**：创建新的功能分支
- **切换分支**：切换到指定分支
- **合并分支**：将一个分支合并到当前分支

### 6. 撤销操作
- **撤销文件修改**：恢复文件到最近提交状态
- **撤销暂存**：取消文件的暂存状态
- **撤销提交**：创建撤销提交（Revert）

### 7. 日志查看
- **提交历史**：显示提交列表（提交Hash、作者、时间、说明）
- **图形化历史**：以图形方式展示分支和合并历史

## 技术架构

### 框架选择
- **Electron**：使用Electron框架构建跨平台桌面应用
- **React**：前端UI框架，提供响应式界面
- **simple-git**：Node.js库，用于执行Git操作
- **TypeScript**：提供类型安全

```

│  标题栏：Git Commander                    [─][□][×] │
├─────────────────────────────────────────────────────┤
│  工具栏：[打开仓库] [创建仓库] [刷新]     当前分支:main │
├─────────────────────────────────────────────────────┤
│         │                                           │

│         │                                           │
├─────────────────────────────────────────────────────┤
```

### 配色方案
- **主色**：#2563EB（蓝色）- 代表科技和专业
- **成功**：#16A34A（绿色）- 已暂存/提交成功
- **警告**：#CA8A04（黄色）- 有修改未提交
- **危险**：#DC2626（红色）- 冲突/删除
- **背景**：#F8FAFC（浅灰）- 主背景
- **文字**：#1E293B（深灰）- 主要文字

### 交互设计
- 所有Git操作都有确认对话框
- 操作结果以Toast通知形式显示
- 加载中的操作显示Loading状态
- 错误信息清晰易懂，不显示技术术语

## 验收标准

### 功能验收
- [ ] 用户可以打开本地Git仓库
- [ ] 用户可以创建新的Git仓库
- [ ] 文件状态正确显示（未跟踪、已修改、已暂存）
- [ ] 用户可以暂存/取消暂存文件
- [ ] 用户可以填写提交信息并完成提交
- [ ] 用户可以推送到远程仓库
- [ ] 用户可以从远程仓库拉取
- [ ] 用户可以查看分支列表
- [ ] 用户可以创建和切换分支
- [ ] 用户可以查看提交历史

###  [ ] 界面响应流畅，无明显体验验收
-卡顿
- [ ] 所有操作都有用户反馈（成功/失败提示）
- [ ] 错误信息友好，不显示技术堆栈
- [ ] 支持快捷键操作（Ctrl+O打开仓库等）

### 稳定性验收
- [ ] 应用启动不崩溃
- [ ] Git操作异常时应用不崩溃
- [ ] 正确处理大仓库（1000+提交）
