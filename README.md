# Git Commander 🚀

一个让 Git 操作变得更简单的图形化工具，为开发者或是Github小白/小懒虫设计，告别繁琐的命令行，一键完成代码管理。

## ✨ 功能亮点

- **可视化操作**：直观的界面设计，无需记忆复杂的 Git 命令
- **一键操作**：支持初始化、拉取、提交、推送等核心功能
- **操作日志**：实时记录操作过程，方便排查问题
- **智能提示**：内置使用指南和常见问题解决方案，降低使用门槛

## 📋 核心功能

| 按钮 | 功能说明 |
|------|----------|
| 🟢 初始化仓库 | 创建新的 Git 仓库 |
| 🟠 一键上传 | 自动完成 `add` → `commit` → `push` 流程 |
| 🔴 一键拉取 | 从远程获取并合并代码 |
| 🔵 一键获取 | 仅获取远程更新，不合并 |
| 🟣 提交更改 | 仅提交到本地仓库 |
| 🔴 克隆仓库 | 从 GitHub 复制仓库到本地 |

## 🚀 快速开始

1. **准备工作**
   - 填写 GitHub 仓库 URL
   - 选择本地文件夹作为工作目录

2. **基本操作流程**
   - 对新仓库：
   ```bash
   1. 在Github上新建仓库，复制仓库链接填入应用中
   2. 选择仓库文件夹，点击初始化仓库
   3. 点击「一键上传」
   ```
   
   - 维护仓库：
   ```bash
   1. 开始工作前：先点击「一键拉取」获取最新代码
   2. 修改代码：在本地进行修改、添加或删除文件
   3. 提交更改：点击「提交更改」或「一键上传」
   4. 推送代码：应用会自动完成 add → commit → push
   ```

本工具旨在简化 Git 工作流，特别适合：
- Git 初学者快速上手
- 频繁进行代码提交的开发者
- 需要可视化操作的团队协作场景
